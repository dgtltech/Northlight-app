## What's new in 1.3

This release works around a macOS 15.4+ change that caused image files to be reported as "damaged" after setting Northlight as their default app via Finder.

### Fixed

- **"File is damaged" when opening images after setting Northlight as default in Finder.** macOS Sequoia 15.4 introduced a Gatekeeper check that blocks files associated through Finder's *Get Info → Open With → Change All…* on anything carrying the quarantine flag (i.e. anything downloaded from Telegram, Safari, AirDrop, Messages, iCloud). Northlight now declares its supported types in a way that no longer trips the related "dangerous types" classification, and the in-app Settings → Defaults screen uses an API that bypasses Finder's broken path entirely.
- **Leica `.rwl` files** now show up correctly in Settings → Defaults (previously listed against the wrong extension).

### New

- **Settings → Defaults → "Fix Quarantine in Folder…"** — pick a folder, Northlight strips the macOS quarantine flag from every supported image inside (recursively). Use this to repair files that already triggered the macOS 15.4+ "damaged" behavior.
- **Information panel in Settings → Defaults** explains the macOS bug, with links to independent technical analyses.
- **Welcome Guide tip** points new users at Settings → Defaults instead of Finder's "Change All…".

### Under the hood

- `Info.plist` document-type declarations split into four logical groups and equipped with `UTImportedTypeDeclarations` for every non-Apple UTI. Improves reliability of *Set as Default* for WebP, AVIF, JPEG XL, PSD, and every RAW vendor format.
- Image loading no longer memory-maps files; switched to plain `Data(contentsOf:)`.
